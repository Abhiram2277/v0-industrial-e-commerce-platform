## Step-by-Step GSC Submission

### Step 1: Verify Domain in GSC
1. Go to [Google Search Console](https://search.google.com/search-console)
2. Click **Add Property**
3. Choose **URL Prefix** (not Domain property initially)
4. Enter: `https://pndindustrialsuppliers.com`
5. Select verification method:
   - **HTML file upload** (Recommended for sites you control)
   - **HTML tag** (Add meta tag to head)
   - **Google Analytics** (If connected)
   - **Google Tag Manager** (If implemented)

### Step 2: Submit Sitemap
Once domain is verified:
1. In GSC, go to **Sitemaps** section
2. Click **Add/test sitemap**
3. Enter the sitemap URL:
   ```
   https://pndindustrialsuppliers.com/sitemap.xml
   ```
4. Click **Submit**
5. Wait for GSC to fetch and validate

### Step 3: Monitor Sitemap Status
In **Sitemaps** section, you can track:
- **Last submitted**: When the sitemap was last processed
- **Last read**: When GSC last crawled it
- **Submitted URLs**: Count of URLs in sitemap
- **Indexed URLs**: Count successfully indexed by Google

### Step 4: Check Indexing Status
Go to **Coverage** report to see:
- **Indexed**: URLs successfully in Google's index
- **Excluded**: URLs intentionally excluded
- **Error**: URLs with indexing issues
- **Valid**: URLs without errors

## Technical Implementation

### Dynamic Generation
The sitemap is auto-generated using Next.js's built-in Sitemap API:
- File: `/app/sitemap.ts`
- Fetches real-time data from products database and blog data
- Updates automatically when content changes
- Serves at: `/sitemap.xml`

### Priority Levels (Explanation)
```
1.0  = Homepage (most important entry point)
0.9  = Blog listing (drives organic traffic)
0.8  = Category pages & About (key navigation)
0.7  = Contact, Quote, Product pages (important for conversions)
0.65 = Individual blog posts (content pages)
```

### Change Frequency
- **Daily**: Homepage (frequent updates)
- **Weekly**: Categories, Products (inventory changes)
- **Monthly**: Blog posts, About (reference content)
- **Monthly**: Static pages (minimal changes)

### Last Modified Dates
- Dynamic pages: Use actual `updatedAt` database timestamps
- Static pages: Updated to current date for recrawl priority
- Blog posts: Use `publishedAt` or `updatedAt` from content

## Robots.txt Configuration

Current configuration in `/public/robots.txt`:
```
User-agent: *
Allow: /
Disallow: /admin
Disallow: /quote?
Allow: /quote

Sitemap: https://pndindustrialsuppliers.com/sitemap.xml
```

**Key Settings**:
- ✓ Allows all regular users (blank User-agent: *)
- ✓ Blocks admin section from crawling
- ✓ Blocks query parameters on quote page (duplicates)
- ✓ Explicitly allows clean quote URLs
- ✓ Points to sitemap for discovery

## Verification & Testing

### Test Sitemap Before Submission
1. Visit directly: `https://pndindustrialsuppliers.com/sitemap.xml`
2. Should see valid XML with `<urlset>` elements
3. Each URL should have:
   - `<loc>`: Full URL
   - `<lastModified>`: ISO 8601 date
   - `<changeFrequency>`: Valid frequency value
   - `<priority>`: Number between 0-1

## Troubleshooting

### Sitemap Not Crawled
- Check robots.txt isn't blocking /sitemap.xml
- Verify domain is verified in GSC
- Wait 24-48 hours for first crawl
- Manually request crawl via GSC

### URLs Not Indexed
- Check individual URLs via URL Inspection Tool
- Verify no noindex tag present
- Check canonical tags point correctly
- Ensure content is unique and valuable

### Coverage Report Issues
- **Excluded**: Check if intentional (e.g., admin pages)
- **Error**: Review specific error messages
- **Indexable but not indexed**: Content quality may be low
  - Improve content quality
  - Add more internal links
  - Ensure good page experience (speed, mobile)
